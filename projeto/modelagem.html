Modelagem de Dados

Estrutura do banco de dados para o sistema de gerenciamento do estúdio de pilates

Visão Geral da Modelagem

A modelagem de dados é uma etapa fundamental no desenvolvimento de sistemas, pois define como as informações serão armazenadas e relacionadas no banco de dados. Para o sistema de gerenciamento do estúdio de pilates, foi desenvolvida uma modelagem completa, embora o foco da implementação inicial seja o CRUD básico de alunos.

Detalhamento das Entidades

Aluno

A entidade Aluno representa os alunos cadastrados no estúdio de pilates.

Atributos:

  • id: Identificador único do aluno (chave primária, auto-incremento)
  • nome: Nome completo do aluno (string, não nulo)
  • email: Email do aluno (string, não nulo, único)
  • telefone: Número de telefone do aluno (string)
  • dataNascimento: Data de nascimento do aluno (date)
  • ativo: Indica se o aluno está ativo no estúdio (boolean, default: true)
  • createdAt: Data de criação do registro (datetime, automático)
  • updatedAt: Data da última atualização do registro (datetime, automático)

Relacionamentos:

  • Um aluno pode participar de várias aulas (1:N com Aula)
  • Um aluno pode realizar vários pagamentos (1:N com Pagamento)

Definição em Sequelize:

// models/Aluno.js
const { Model, DataTypes } = require('sequelize');
const sequelize = require('../config/database');

class Aluno extends Model {}

Aluno.init(
  {
    id: {
      type: DataTypes.INTEGER,
      primaryKey: true,
      autoIncrement: true,
    },
    nome: {
      type: DataTypes.STRING,
      allowNull: false,
    },
    email: {
      type: DataTypes.STRING,
      allowNull: false,
      unique: true,
      validate: {
        isEmail: true,
      },
    },
    telefone: {
      type: DataTypes.STRING,
    },
    dataNascimento: {
      type: DataTypes.DATEONLY,
    },
    ativo: {
      type: DataTypes.BOOLEAN,
      defaultValue: true,
    },
  },
  {
    sequelize,
    modelName: 'aluno',
    timestamps: true,
  }
);

module.exports = Aluno;
Instrutor

A entidade Instrutor representa os profissionais que ministram aulas no estúdio de pilates.

Atributos:

  • id: Identificador único do instrutor (chave primária, auto-incremento)
  • nome: Nome completo do instrutor (string, não nulo)
  • email: Email do instrutor (string, não nulo, único)
  • telefone: Número de telefone do instrutor (string)
  • especialidade: Área de especialização do instrutor (string)
  • ativo: Indica se o instrutor está ativo no estúdio (boolean, default: true)
  • createdAt: Data de criação do registro (datetime, automático)
  • updatedAt: Data da última atualização do registro (datetime, automático)

Relacionamentos:

  • Um instrutor pode ministrar várias aulas (1:N com Aula)

Nota: Esta entidade não será implementada na versão inicial do CRUD.

Modalidade

A entidade Modalidade representa os tipos de aulas oferecidas no estúdio de pilates.

Atributos:

  • id: Identificador único da modalidade (chave primária, auto-incremento)
  • nome: Nome da modalidade (string, não nulo)
  • descricao: Descrição detalhada da modalidade (text)
  • duracao: Duração da aula em minutos (integer)
  • capacidade: Número máximo de alunos por aula (integer)
  • createdAt: Data de criação do registro (datetime, automático)
  • updatedAt: Data da última atualização do registro (datetime, automático)

Relacionamentos:

  • Uma modalidade pode ter várias aulas (1:N com Aula)

Nota: Esta entidade não será implementada na versão inicial do CRUD.

Aula

A entidade Aula representa as sessões de aula agendadas no estúdio de pilates.

Atributos:

  • id: Identificador único da aula (chave primária, auto-incremento)
  • data: Data da aula (date, não nulo)
  • horario: Horário da aula (time, não nulo)
  • alunoId: ID do aluno (chave estrangeira, não nulo)
  • instrutorId: ID do instrutor (chave estrangeira, não nulo)
  • modalidadeId: ID da modalidade (chave estrangeira, não nulo)
  • status: Status da aula (enum: 'agendada', 'concluída', 'cancelada', default: 'agendada')
  • observacoes: Observações sobre a aula (text)
  • createdAt: Data de criação do registro (datetime, automático)
  • updatedAt: Data da última atualização do registro (datetime, automático)

Relacionamentos:

  • Uma aula pertence a um aluno (N:1 com Aluno)
  • Uma aula é ministrada por um instrutor (N:1 com Instrutor)
  • Uma aula é de uma modalidade específica (N:1 com Modalidade)

Nota: Esta entidade não será implementada na versão inicial do CRUD.

Pagamento

A entidade Pagamento representa os pagamentos realizados pelos alunos.

Atributos:

  • id: Identificador único do pagamento (chave primária, auto-incremento)
  • alunoId: ID do aluno (chave estrangeira, não nulo)
  • valor: Valor do pagamento (decimal, não nulo)
  • dataPagamento: Data em que o pagamento foi realizado (date, não nulo)
  • metodoPagamento: Método utilizado para o pagamento (enum: 'dinheiro', 'cartão', 'pix', 'transferência')
  • status: Status do pagamento (enum: 'pendente', 'pago', 'cancelado', default: 'pendente')
  • createdAt: Data de criação do registro (datetime, automático)
  • updatedAt: Data da última atualização do registro (datetime, automático)

Relacionamentos:

  • Um pagamento pertence a um aluno (N:1 com Aluno)

Nota: Esta entidade não será implementada na versão inicial do CRUD.

Scripts SQL

Abaixo estão os scripts SQL para criação das tabelas no banco de dados MySQL:

Criação do Banco de Dados

-- Criação do banco de dados
CREATE DATABASE IF NOT EXISTS pilates_studio;
USE pilates_studio;

Tabela de Alunos

-- Tabela de Alunos
CREATE TABLE IF NOT EXISTS alunos (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(255) NOT NULL,
  email VARCHAR(255) NOT NULL UNIQUE,
  telefone VARCHAR(20),
  data_nascimento DATE,
  ativo BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

Nota: Os scripts para as demais tabelas não serão implementados na versão inicial do CRUD.

Considerações sobre a Modelagem

A modelagem apresentada é completa e abrange todas as entidades necessárias para um sistema de gerenciamento de estúdio de pilates. No entanto, para a implementação inicial do CRUD, focaremos apenas na entidade Aluno, que é a base do sistema.

As demais entidades (Instrutor, Modalidade, Aula e Pagamento) estão documentadas para referência futura e podem ser implementadas em versões posteriores do sistema, à medida que os alunos avançam no curso e adquirem mais conhecimentos.

Esta abordagem permite que os alunos iniciantes se concentrem em aprender os conceitos básicos de CRUD com Node.js e Express, sem se sobrecarregar com a complexidade de um sistema completo.
