Módulo 8: Metodologias Ágeis e Scrum

Princípios e práticas para desenvolvimento de software colaborativo e eficiente

Visão Geral do Módulo

Neste módulo, você aprenderá sobre metodologias ágeis de desenvolvimento de software, com foco especial no framework Scrum. Você conhecerá os princípios, valores e práticas que permitem às equipes entregar software de qualidade de forma iterativa e incremental, respondendo rapidamente às mudanças e priorizando a colaboração e a entrega de valor.

Carga Horária

8 horas

Nível

Intermediário

Pré-requisitos

Módulos 1, 2, 3, 4, 5, 6 e 7

Exemplos Práticos

Exemplo de Product Backlog para o Sistema de Gerenciamento de Estúdio de Pilates

ID História de Usuário Critérios de Aceitação Prioridade Estimativa (Story Points)
US-01 Como administrador, quero cadastrar novos alunos no sistema para manter um registro atualizado dos clientes do estúdio.
  • Deve ser possível cadastrar nome, email, telefone, data de nascimento e endereço
  • O email deve ser único no sistema
  • Todos os campos devem ser validados
  • Deve exibir mensagem de sucesso após o cadastro
 Alta 5
US-02 Como administrador, quero visualizar a lista de todos os alunos cadastrados para ter uma visão geral dos clientes do estúdio.
  • A lista deve mostrar nome, email e telefone de cada aluno
  • Deve ser possível ordenar por nome, data de cadastro
  • Deve ser possível filtrar alunos ativos/inativos
  • Deve ser possível pesquisar por nome ou email
 Alta 3
US-03 Como administrador, quero editar as informações de um aluno para manter os dados atualizados.
  • Deve ser possível editar todos os campos do cadastro
  • Deve validar os dados antes de salvar
  • Deve exibir mensagem de sucesso após a atualização
 Alta 3
US-04 Como administrador, quero desativar o cadastro de um aluno quando ele parar de frequentar o estúdio.
  • Deve solicitar confirmação antes de desativar
  • Deve permitir reativar um aluno desativado
  • Alunos desativados não devem aparecer na lista principal por padrão
 Média 2
US-05 Como administrador, quero visualizar os detalhes completos de um aluno para ter acesso a todas as suas informações.
  • Deve mostrar todos os dados cadastrais
  • Deve mostrar histórico de aulas
  • Deve mostrar histórico de pagamentos
  • Deve permitir navegar para as telas de edição
 Média 3
US-06 Como administrador, quero cadastrar instrutores no sistema para registrar quem pode ministrar aulas no estúdio.
  • Deve ser possível cadastrar nome, email, telefone, especialidade
  • O email deve ser único no sistema
  • Todos os campos devem ser validados
 Alta 5
US-07 Como administrador, quero cadastrar modalidades de aulas para organizar os diferentes tipos de atividades oferecidas.
  • Deve ser possível cadastrar nome, descrição, duração padrão
  • O nome deve ser único no sistema
 Alta 3
US-08 Como administrador, quero agendar aulas no sistema para organizar a programação do estúdio.
  • Deve ser possível selecionar modalidade, instrutor, data/hora, capacidade
  • Não deve permitir conflito de horários para o mesmo instrutor
  • Deve validar disponibilidade de sala
 Alta 8

Exemplo de Sprint Planning para o Sistema de Gerenciamento de Estúdio de Pilates

Sprint 1: Cadastro e Gestão de Alunos

Objetivo do Sprint: Implementar as funcionalidades básicas de cadastro e gestão de alunos.

Duração: 2 semanas

Sprint Backlog:
  • US-01: Cadastro de alunos
    • Tarefa 1: Criar modelo de dados para alunos (2h)
    • Tarefa 2: Implementar API para criação de alunos (4h)
    • Tarefa 3: Criar formulário de cadastro no frontend (6h)
    • Tarefa 4: Implementar validações de formulário (3h)
    • Tarefa 5: Testar e corrigir bugs (3h)
  • US-02: Listagem de alunos
    • Tarefa 1: Implementar API para listar alunos (3h)
    • Tarefa 2: Criar interface de listagem no frontend (4h)
    • Tarefa 3: Implementar filtros e ordenação (5h)
    • Tarefa 4: Testar e corrigir bugs (2h)
  • US-03: Edição de alunos
    • Tarefa 1: Implementar API para atualização de alunos (3h)
    • Tarefa 2: Criar formulário de edição no frontend (4h)
    • Tarefa 3: Implementar validações (2h)
    • Tarefa 4: Testar e corrigir bugs (2h)
  • US-04: Desativação de alunos
    • Tarefa 1: Implementar API para ativar/desativar alunos (2h)
    • Tarefa 2: Criar interface para confirmação (2h)
    • Tarefa 3: Atualizar listagem para filtrar por status (2h)
    • Tarefa 4: Testar e corrigir bugs (1h)
Capacidade do Time:
  • 3 desenvolvedores x 30 horas produtivas por semana x 2 semanas = 180 horas
  • Total de horas estimadas: 48 horas
  • Capacidade restante: 132 horas (para outras tarefas, reuniões, documentação, etc.)
Definição de Pronto:
  • Código implementado e revisado
  • Testes unitários implementados com cobertura mínima de 80%
  • Testes de integração implementados
  • Documentação atualizada
  • Funcionalidade aprovada pelo Product Owner

Exercícios Práticos

Para fixar os conceitos aprendidos neste módulo, recomendamos a realização dos seguintes exercícios:

  1. Crie um Product Backlog para um projeto de sua escolha, com pelo menos 10 histórias de usuário.
  2. Escreva critérios de aceitação detalhados para 3 histórias de usuário.
  3. Realize uma sessão de Planning Poker para estimar histórias de usuário (em grupo).
  4. Planeje um Sprint completo, incluindo Sprint Backlog e quebra de tarefas.
  5. Simule uma Daily Scrum com sua equipe ou colegas.
  6. Conduza uma Sprint Review simulada para apresentar um incremento de produto.
  7. Realize uma Sprint Retrospective para identificar pontos de melhoria em um projeto.
  8. Configure um quadro Kanban (físico ou digital) para acompanhar o progresso das tarefas.

Você encontrará mais exercícios e desafios na seção de Exercícios.
